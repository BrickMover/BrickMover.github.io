Java集合类

Set

集合类分为两个大类：Collection和Map，区别在于每个位置保存的元素个数，Collection保存的是一个元素，Map保存的是一个键值对。

1. Collection

根据特性，Collection主要有Set，Queue，List三部分组成

1.1 Set

不可重复，无序的集合，主要继承关系

1.1.1 HashSet

HashSet 的实现其实非常简单，它只是封装了一个 HashMap 对象来存储所有的集合元素，所有放入 HashSet 中的集合元素实际上由 HashMap 的 key 来保存，而 HashMap 的 value 则存储了一个 PRESENT，它是一个静态的 Object 对象。

1.1.2 LinkedHashSet

HashSet 的不同之处在于，LinkedHashSet 维护着一个运行于所有条目的双重链接列表。此链接列表定义了迭代顺序，该迭代顺序可为插入顺序或是访问顺序。

1.1.3 SortedSet

该SortedSet的接口扩展集Set并声明了一组按升序排序的行为。除了通过集定义的那些方法中，SortedSet接口声明总结于下表的方法：

SN 方法描述
1 Comparator comparator( )返回调用有序set的比较。如果自然顺序用于此set，则返回null。
2 Object first( )返回调用排序集合的第一个元素。
3 SortedSet headSet(Object end)返回包含包含在调用的有序set的元素小于结束一个SortedSet。在返回的有序set的元素也被调用的有序set引用。
4 Object last( )返回调用排序集合的最后一个元素。
5 SortedSet subSet(Object start, Object end)返回一个SortedSet，包括开始和end-1之间的那些元素。被返回的集合中的元素也被调用对象引用。
6 SortedSet tailSet(Object start)返回包含大于或等于启动包含在有序集合这些元素的SortedSet。在返回的集合中的元素也被调用对象的引用。

1.1.4 EnumSet

EnumSet是一个与Enum一起使用的Set，这个Set中的元素必须来自单个枚举类型。
枚举类型在Set创建时显式或者隐式创建，枚举set在内部表示位向量，空间和时效性能都很好。
几个特点

1.由iterator方法返回的迭代器按其自然顺序遍历这些元素（该顺序是声明枚举常量的顺序）。返回的迭代器是弱一致的：它从不抛出ConcurrentModificationException，也不一定显示在迭代进行时发生的任何set修改的效果。

2.不允许使用null元素。插入null元素将抛出NullPointerException。但是，判断是否存在null元素或移除null元素将不会抛出异常。

3.像大多数collection一样，EnumSet是不同步的。如果多个线程同时访问一个枚举set，需要在外部控制同步。

4.Enum的枚举常量的位置(序数)可以用Enum的ordinal()方法得到.

EnumSet的创建

EnumSet将枚举值保存在一个长整型数据上，每个枚举值占用1bit。

与RegularEnumSet不同的是JumboEnumSet将其放在一个长整型数组上进行操作。

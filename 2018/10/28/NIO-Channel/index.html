1 Channel定义

全新的，极好的Java IO示例，提供与IO服务的直接连接。Channel用于在buffer和channel另外一侧的obj(通常是file或者socket)

不同操作系统上的Channel实现会有根本上的差异

1.1 打开Channel

IO可以分为File IO和Stream IO，Channel也是如次分为FileChannel和三个socket channel(SocketChannel, ServerSocketChannel, DatagramChannel).

Channel不同于Stream是，CHannel可以选择是单向还是双向，而Stream只能是单向通道

1.1.1 FileChannel open

FileChannel只能通过一个打开的RandomAccessFile，FileInputStream或FileOutputStream对象上使用getChannel
文件通道总是阻塞式的，因此不能用于非阻塞模式

FileChannel对象是线程安全的

2 FileChannel相关方法

FileChannel抽象类源码

2.1 position

无参的position函数，返回当前文件的position
有参的position函数，返回指定位置的channel对象，负值会导致java.lang.IllegalArgumentException异常，数值越界时，读返回文件尾，写则造成文件空洞

2.2 size

size方法返回关联文件的大小
long fileSize = channel.size()

2.3 truncate

截取文件方法，将指定长度后面的部分删除，保留指定长度部分
channel.truncate(1024)截取前1024字节

2.4 force

该方法告诉通道强制将全部待定的修改都应用到磁盘的文件上。所有的现代文件系统都会缓存数据和延迟磁盘文件更新以提高性能。调用force()方法要 求文件的所有待定修改立即同步到磁盘。channel.force(true)

3 ServerSocketChannel

抽象类源码

ServerSocketChannel 是一个基于通道的 socket 监听器。它同我们所熟悉的java.net.ServerSocket执行相同的基本任务，不过它增加了通道语义，因此能够在非阻塞模式下运行。

3.1 打开ServerSocketChannel

ServerSocketChannel ssc = ServerSocketChannel.open();返回一个未绑定的java.net.ServerSocket关联的通道。该ServerSocket需要通过方法socket()获取

3.2 accept

同它的对等体java.net.ServerSocket一样，ServerSocketChannel也有accept()方法。

如果在ServerSocket上调用accept()方法，那么它会同任何其他的ServerSocket表现一样:总是阻塞并返回一个java.net.Socket对象。

如果您选择在ServerSocketChannel上调用accept()方法则会返回SocketChannel类型的对象，返回的对象能够在非阻塞模式下运行。

打开一个SocketChannel，链接到互联网上的某台机器
